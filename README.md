# chinese-pos-classifier（汉语词类划分器）

一个给 Claude 用的 [Agent Skill](https://docs.claude.com/en/docs/agents-and-tools/agent-skills/overview)，用于对汉语语素组（候选词、单字、词语）做成词判断和词类归类，依据北大版现代汉语词类标注标准，支持兼类词识别，并对判断依据不充分的模糊案例明确标注、列出多种可能归类及各自理由，而不是强行给出单一答案。

## 这个 skill 做什么

给定一个或多个中文语素组，Claude 会依次：

1. **成词判断**——判断这个语素组是否是一个词（而非黏着语素或临时组合），以词典收录为主、句法测试为辅。
2. **大类判断**——定位到体词 / 谓词 / 饰词 / 虚词 / 特殊词类五大类中的哪一类。
3. **细类判断**——用一套操作性的快速判别测试（如"能否受'很'修饰""能否带'了/着/过+宾语'"）确定具体词类。
4. **兼类判断**——核心标准是词义是否发生**转指**，而不是能否出现在不同句法位置，避免过度或漏判兼类。
5. **模糊地带标注**——测试结果矛盾、或学界本身有争议的情况，会明确列出所有可能归类及理由，而不是假装确定。

输出是一段简洁的文字说明，不用表格，可以一次处理一个词，也可以一次给多个词逐一判断。

## 依据的标准

词类体系和判断标准整理自使用者的现代汉语语法课程材料（教材标准原文 + 课堂总结的判断流程 + 老师提供的标准例词答案），核心是北大版的四大类词类体系（体词/谓词/饰词/虚词，外加代词/拟声词/叹词三个按语义功能划分的特殊词类）。如果你用的是别的词类标注标准（比如宾州中文树库标注集、其他院校的语法教材），词类定义和判别测试可能对不上，需要自行替换 `references/word_classes.md` 和 `references/decision_heuristics.md` 里的内容。

## 目录结构

```
chinese-pos-classifier/
├── SKILL.md                          # 主判断流程和输出格式
└── references/
    ├── word_classes.md               # 每个词类的完整定义、语法特点、原始论证
    ├── decision_heuristics.md        # 快速判别测试 + 兼类判断方法论
    └── calibration_examples.md       # 已有标准答案的例词，用于校准判断方法
```

## 使用示例

```
用户：帮我判断"温暖"这个词的词类，有没有兼类。

Claude（自动触发本 skill）：
"温暖"是词典收录的常规词，成词没有疑问，是一个兼类词：
表"温暖"这个性质时是形容词（很温暖、天气温暖），做主语时
（"温暖是这个季节的主题"）仍然指这个性质本身，没有转指，
不算名词兼类；但做谓语表示"使人变得温暖"这个动作时
（"这句话温暖了我"），词义从性质转指为致使动作，判定为
形容词 + 动词兼类。
```
